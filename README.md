The system operates by implementing a recursive backtracking engine that navigates a 2D integer array representing the 3×3 Sudoku grid. When the program initiates, it performs a character-by-character scan of the user's input to populate the matrix, immediately triggering validation checks to ensure each entry is a valid digit between 1 and 3 or an empty placeholder.

During the solving phase, the algorithm employs a depth-first search strategy, identifying empty cells and systematically attempting to place numbers. Before each placement, it invokes a constraint-checking function that performs a linear scan of the current row and column to verify that the target digit does not already exist, satisfying the fundamental Sudoku rules. If a placement leads to an unsolvable state, the recursion "unwinds," clearing the cell and reverting to the previous state to try a different numerical path. This automated search continues until the grid is either fully populated or determined to be mathematically impossible.
